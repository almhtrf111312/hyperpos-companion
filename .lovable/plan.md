

# إصلاح ظهور شروط الخصوصية مرتين

## المشكلة
عند تسجيل الدخول، الكود في `use-auth.tsx` يمسح جميع مفاتيح `hyperpos_*` من localStorage (ما عدا اللغة والثيم). هذا يشمل مسح مفتاح `hyperpos_privacy_accepted`، مما يسبب ظهور شاشة الخصوصية مرة ثانية بعد تسجيل الدخول.

## الحل
إضافة `hyperpos_privacy_accepted` إلى قائمة المفاتيح المحمية من المسح في مكانين:

### التعديل في `src/hooks/use-auth.tsx`:

**1. في دالة `signIn` (سطر 334-338):** إضافة استثناء لمفتاح الخصوصية:
```
key !== 'hyperpos_privacy_accepted'
```

**2. في دالة `signOut` (سطر 463-466):** نفس الإضافة:
```
key !== 'hyperpos_privacy_accepted'
```

## النتيجة
- شاشة الخصوصية تظهر مرة واحدة فقط عند أول فتح للتطبيق
- لا تظهر مرة أخرى بعد تسجيل الدخول أو الخروج
