

# نقل إدارة الأكواد وتشخيص النظام إلى لوحة البوس

## ملخص التغييرات

### 1. تبسيط "إدارة التراخيص" في الإعدادات
تبديل محتوى تبويب "إدارة التراخيص" في صفحة الإعدادات ليعرض فقط زر "فتح لوحة البوس" مع طلب كلمة مرور الحساب قبل الانتقال (كعامل أمان إضافي).

### 2. إزالة تبويب "تشخيص النظام" من الإعدادات
حذف تبويب `diagnostics` من قائمة تبويبات الإعدادات بالكامل.

### 3. إضافة تشخيص النظام إلى لوحة البوس
إضافة مكون `SystemDiagnostics` داخل صفحة البوس كقسم جديد.

---

## التفاصيل التقنية

### الملف: `src/pages/Settings.tsx`

**تغيير 1 - حذف تبويب التشخيص من القائمة (سطر 185):**
حذف السطر:
```
{ id: 'diagnostics', label: t('settings.diagnostics'), icon: Wrench, bossOnly: true },
```

**تغيير 2 - تبسيط تبويب licenses (سطر 1471-1487):**
استبدال المحتوى الحالي بـ:
- عنوان ووصف بسيط
- حقل إدخال كلمة المرور
- زر "فتح لوحة البوس" يتحقق من كلمة المرور عبر `supabase.auth.signInWithPassword` قبل الانتقال
- عند نجاح التحقق، ينتقل إلى `/boss`

**تغيير 3 - حذف case 'diagnostics' (سطر 935-940):**
حذف الـ case بالكامل من دالة `renderTabContent`.

### الملف: `src/pages/BossPanel.tsx`

**تغيير 1 - إضافة import لمكون SystemDiagnostics:**
```typescript
import { SystemDiagnostics } from '@/components/settings/SystemDiagnostics';
```

**تغيير 2 - إضافة قسم تشخيص النظام:**
إضافة قسم جديد في نهاية محتوى الصفحة يحتوي على مكون `SystemDiagnostics` داخل Card.

---

## سير العمل الجديد

1. المستخدم يفتح الإعدادات ويضغط "إدارة التراخيص"
2. يظهر له فقط: عنوان + حقل كلمة مرور + زر "فتح لوحة البوس"
3. يدخل كلمة المرور ويضغط الزر
4. النظام يتحقق من كلمة المرور عبر الخادم
5. عند النجاح ينتقل إلى لوحة البوس التي تحتوي على كل شيء: إدارة الحسابات، إدارة الأكواد، تشخيص النظام
